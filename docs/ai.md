**目的:**
最新安定版の Next.js (App Router 使用) と、**指定されたバージョンの Chakra UI (バージョン 2.8.2)** を用いた、ベストプラクティスに沿ったアプリケーション開発を学ぶための AI 駆動開発プロジェクト。API やデータベースを使わない、1 画面で完結する最もシンプルな構成のカウントアップアプリを実装する。

**技術スタック:**

- 最新安定版の Next.js (App Router)
- **Chakra UI (バージョン 2.8.2 を使用)**

**機能要件:**

1.  画面中央に現在のカウント数を大きく表示する。初期値は 0 とする。
2.  カウント数を 1 増やす「Increment」ボタンを配置する。
3.  カウント数を 0 に戻す「Reset」ボタンを配置する。
4.  アプリケーションは 1 画面のみで、ページ遷移は発生しない。

**技術要件・制約:**

- API、データベース、外部ストレージは一切使用しない。
- カウント数の状態管理は、React の`useState`など、クライアントサイドの基本的な状態管理機能のみで行う。
- Next.js は App Router (`app`ディレクトリ) 構成を使用する。最新安定版を想定。
- **Chakra UI はバージョン 2.8.2 を使用する。**
- 生成されるコードは、Next.js App Router および指定されたバージョンの Chakra UI のベストプラクティスに沿った、可読性が高く再利用しやすい**模範的なコード**であることを強く意識すること。
- 必要最低限のコード量で、かつ機能的・構造的に分かりやすく記述すること。

**UI/UX 要件:**

- Chakra UI の設計思想である**「アクセシブルでシンプル」「柔軟で一貫性のある」**デザインを最優先とする。
- モダンで洗練されたデザイン（Chakra UI の標準コンポーネントを効果的に使用）とする。
- **優先度は高くない**が、Google: Material Design や Apple: Human Interface Guidelines で提唱されている一般的な UI/UX 原則（例: 要素間の適切なスペーシング、分かりやすい視覚的階層、直感的な操作感など）を意識した、ユーザーフレンドリーなデザインとする。
- PC およびスマートフォン双方でレイアウトが崩れない、基本的なレスポンシブ対応。Chakra UI のレスポンシブスタイリング機能を活用すること。
- ボタンは分かりやすく配置し、視覚的に操作しやすいようにする。
- カウント数は視認性の高いフォントサイズで表示する。
